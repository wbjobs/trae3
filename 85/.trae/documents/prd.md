## 1. 产品概述

战术推演指挥系统 —— 一套面向军事爱好者和战术训练场景的客户端/服务端分离式兵棋推演平台。客户端提供沉浸式阵地视图渲染与实时操作交互，服务端负责战术引擎推演、跨端状态同步、对局存档与配置数据库对接。

- 目标用户：军事院校训练人员、兵棋爱好者、战术研究团队
- 核心价值：将传统兵棋推演数字化，实现多人实时协作战术模拟，降低推演组织成本，提高训练效率

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 指挥官 | 邀请码注册 | 创建对局、部署兵力、发起指令、查看推演结果 |
| 参谋 | 邀请码注册 | 加入对局、查看态势、提交建议、标记目标 |
| 观察员 | 开放注册 | 观看推演过程、查看历史回放 |

### 2.2 功能模块

1. **战情大厅**：对局列表、快速匹配、创建房间、战绩统计
2. **阵地视图**：2D 战术地图渲染、网格坐标系、地形图层叠加、兵力标识渲染
3. **指挥面板**：兵力部署、移动指令、攻击指令、防御指令、指令队列管理
4. **推演引擎**：回合制推演、实时推演模式切换、战损计算、态势演变动画
5. **态势同步**：多端实时状态同步、操作日志广播、冲突仲裁
6. **数据存档**：对局回放、战报导出、统计分析

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|

## 3. 核心流程

**对局全流程**：指挥官在大厅创建房间 → 配置地图与参数 → 参谋加入选择阵营 → 进入阵地视图部署初始兵力 → 指挥官下达作战指令 → 服务端推演引擎执行战术计算 → 结果同步至所有客户端 → 态势动画播放 → 进入下一回合/时间步 → 对局结束 → 自动存档并生成战报

    "结果同步至所有客户端" --> "态势动画播放"

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：军绿色 (#2D4A22) 搭配暗沙色 (#C4A35A) 作为点缀，深炭灰 (#1A1A2E) 为背景底色
- **辅助色**：红方 (#C62828)、蓝方 (#1565C0) 用于阵营区分，琥珀色 (#FF8F00) 用于警告/高亮
- **按钮风格**：硬朗矩形 + 微圆角 (2px)，按压态下沉阴影，军事风格边框
- **字体**：标题使用 Noto Sans SC Bold，正文使用 Noto Sans SC Regular，数据使用 JetBrains Mono
- **布局风格**：左侧战术地图为主区域（占比 70%），右侧指挥面板为辅助区域（占比 30%），顶部状态栏
- **图标风格**：线框军事图标 (NATO 标准军标风格)，扁平化但带等距网格质感

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 战情大厅 | 对局列表 | 卡片网格布局，每张卡片含地图缩略图/房间名/玩家数/状态标签，深色毛玻璃背景 |
| 战情大厅 | 创建房间 | 模态对话框，表单含地图选择器/模式切换/兵力滑块/时间输入，军事绿主按钮 |
| 阵地视图 | 战术地图 | Canvas 全区域渲染，六角网格叠加，缩放控制条在左下角，坐标信息右上角浮层 |
| 阵地视图 | 兵力标识 | SVG 军标图标，阵营色圆环，悬停展开属性面板，选中态发光描边 |
| 指挥面板 | 指令输入 | 右侧面板，上方单位选择器，中间地图指令预览线，下方指令按钮组 |
| 指挥面板 | 指令队列 | 列表式排列，每条指令含图标/描述/删除按钮，底部提交按钮带倒计时 |
| 推演引擎 | 态势动画 | Canvas 逐帧动画，单位平滑移动轨迹线，交火粒子特效，战损数字上浮消失 |
| 态势同步 | 操作日志 | 右下角半透明浮层，消息滚动列表，按阵营着色，新消息高亮淡出 |
| 数据存档 | 回放系统 | 底部时间轴拖拽条，播放/暂停/速度按钮，进度浮层显示当前回合数 |
| 数据存档 | 战报导出 | 全屏模态，左侧统计图表(折线图/饼图)，右侧事件时间线，顶部导出按钮组 |

### 4.3 响应式策略

- 桌面优先设计，战术地图核心区域最低宽度 960px
- 平板端指挥面板折叠为底部抽屉
- 移动端暂不支持完整操作，仅提供观察员视角

### 4.4 3D 场景指引

本项目为 2D 战术地图，不涉及 3D 场景。通过 Canvas 2D 渲染配合阴影与渐变营造立体等距视觉感。
